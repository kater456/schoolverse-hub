# Campus Market Major Upgrade Plan

This is a large, multi-area change. I'll break it into phases and ship them in one pass.

## Phase 1 — Database (single migration)

New tables (all with GRANTs + RLS):

- **`reel_likes`** — `reel_id uuid`, `user_id uuid null`, `session_id text null`, `created_at`. Anon can insert/select; one like per (reel, user_id) or (reel, session_id).
- **`vendor_followers`** — `vendor_id uuid`, `user_id uuid`, unique pair. Auth users insert/delete their own; vendor can read own followers; anyone can read counts.
- **`scheduled_pickups`** — `vendor_id`, `user_id`, `pickup_at timestamptz`, `note text`, `status text default 'pending'`. Buyer manages own; vendor sees own.

## Phase 2 — Scroll fix

- Create `src/components/ScrollManager.tsx` (smooth scroll-to-top with 10 ms delay on `pathname` change).
- Mount it in `App.tsx` inside `BrowserRouter`, replacing/augmenting existing `ScrollToTop`.
- In `src/main.tsx`, set `history.scrollRestoration = 'manual'` before render.

## Phase 3 — Vendor Profile redesign (`src/pages/VendorProfile.tsx`)

Sections (top → bottom):

1. **Header** — banner + avatar, name, category tag, verified badge, Open/Closed pill (computed from `business_hours` if present, fallback Open), distance using existing live-location lat/lng + browser geolocation, stat row (sales, rating, followers).
2. **Promo banner** — fire-emoji animated banner if `vendor_deals` active.
3. **Tabs** — `Products | Reels | Reviews`.
   - Products: horizontal swipe cards with `+ Cart` and ⚡ DEAL badge if deal active.
   - Reels: vertical feed of `vendor_videos`, like button with heart burst animation backed by `reel_likes`.
   - Reviews: star breakdown bars, recent list, "Write Review" CTA (login required), "X students bought this week" social proof from `transactions`.
4. **Sticky bottom action bar (mobile)** — Message, Follow/Unfollow, Quick Cart sheet, Schedule Pickup sheet, Share (Web Share API), Quick Question floating bubble with prefilled chips.

New small components under `src/components/vendor/`:
- `VendorProfileHeader.tsx`

## Phase 4 — Global promo toast

- New `src/components/PromoToastListener.tsx` — Supabase realtime subscription to `vendor_deals` inserts where `is_active=true`. Shows sonner toast "🔥 [Vendor] is running a promo! Tap to view" for 6 s, links to `/vendor/:id`.
- Mount only on `/` and `/browse` (conditional inside component using `useLocation`).

## Phase 5 — AI Chatbox improvements (`src/components/AIChatbox.tsx` + `supabase/functions/ai-chat/index.ts`)

- Tighten system prompt: short, friendly, campus-context aware (vendors, promos, registration, payments, delivery). When unsure → reply "Let me connect you to the vendor directly 👉" and include a `[CONNECT:vendor_id]` token the UI detects to render a button opening `/chat` or vendor profile.
- Edge function returns `{ reply, suggestions: string[] }`. Suggestions generated by asking the model for 3 short follow-ups; fall back to defaults.
- UI: render markdown via `react-markdown`, render quick-reply chips below each assistant message, refined bouncing-dots typing indicator (already in place; polish).
- Allow chatbox for logged-out users too (remove the `if (!user) return null` guard).

## Phase 6 — Wiring/cleanup

- Update `src/App.tsx`: `<ScrollManager />` inside Router, `<PromoToastListener />` near other global components.
- Keep existing dark navy + orange tokens; use `bg-accent`, `text-primary`, etc.
- Mobile-first; `animate-fade-in`, `animate-scale-in`, `hover-scale` utilities for smoothness.

## Notes / non-goals

- Distance uses existing `vendor_presence.live_location_lat/lng` only when the vendor has it on; otherwise hidden.
- Open Now uses any existing `business_hours` JSON column on `vendors`; if absent, falls back to "Open".
- Quick Cart uses existing `useCart` hook.
- Share uses `navigator.share` with fallback to copy-to-clipboard.
